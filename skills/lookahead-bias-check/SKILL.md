---

# 目的
実装済みコードや result_summary を読み、未来データ利用やデータリークの疑いを洗い出す。

# 使う場面
- 新戦略の初回レビュー
- 特徴量を追加した直後
- 成績が急によくなったとき

# チェック項目
1. 特徴量に未来の価格や future return が入っていないか
2. 当日終値で作ったシグナルを当日寄りや当日中に約定していないか
3. ラベル生成と特徴量生成の時点分離が壊れていないか
4. 学習期間と評価期間の境界が崩れていないか
5. 前処理が全期間 fit になっていないか

# 出力形式
- 問題なし / 要確認 / 問題あり
- 根拠
- 修正提案
