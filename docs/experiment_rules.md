# experiment_rules.md

## 1. 実験の考え方
この研究は「単発の正解探し」ではなく、「比較可能な仮説の反復改善」を目的とする。

## 2. 1 worker 1 仮説
各 worker は1つの中心仮説を持つ。
複数要素を混ぜる場合も、何が主因か説明できる粒度に抑える。

## 3. 仮説の作り方
良い仮説の条件:
- なぜ効くかの機序が説明できる
- 必要データが明確
- エントリー/イグジットへ落とせる
- 過学習しにくい

## 4. 変更の優先順位
改善時は次の順で考える。
1. 先読みや実装ミスの除去
2. ルールの簡素化
3. リスク管理の改善
4. 特徴量の追加
5. モデル複雑化

## 5. 比較ルール
- 同じデータ条件で比較する
- 手数料・スリッページ条件を揃える
- ベンチマークと同じ評価期間で比べる
- 指標は複数見る

## 6. 廃棄基準
以下は原則として次回候補から外す。
- 先読み混入の疑いが強い
- 取引がほぼ発生しない
- ベンチマークより一貫して劣後
- ルールが複雑すぎ再現しづらい

## 7. 採用基準
次回へ引き継ぐ候補は以下。
- 仮説の機序が説明可能
- 検証結果が安定している
- リスク調整後で優位性の兆候がある
- 実装コストに対して改善余地がある

## 8. summary_writer への要求
- ただ順位を出すだけでは不十分
- 「どの仮説が効いたか」「なぜ弱かったか」を書く
- 次に統合すべき改善点を具体化する

## 9. 次世代 worker の作り方
新しい worker は、前回の優れた点を統合して作る。
単純コピーではなく、以下を記述する。
- 引き継ぐ要素
- 捨てる要素
- 新規追加する要素
- 期待する改善点
