# strategy_catalog.md

## benchmark_buy_and_hold
- `^N225` をテスト期間で単純保有する比較用ベンチマーク。
- 戦略の優劣ではなく、同期間の地合いを把握するために使う。

## worker_01
- 出来高増加を伴うブレイクアウトを狙うルールベース戦略。
- 強い上昇の初動を取りやすい一方、だましの上抜けに弱い。

## worker_02
- 短期の下落後に入る自律反発を狙うルールベース戦略。
- 短期反転には強いが、下落トレンド継続局面では被弾しやすい。

## worker_03
- 低ボラティリティ収縮後の値幅拡大を狙うイベント型戦略。
- 伸びる日は大きいが、イベント失敗時のぶれも大きい。

## worker_04
- `LogisticRegression` による線形 ML 戦略。
- 安定した順位付けに向く一方、非線形な局面変化は拾いにくい。

## worker_05
- `HistGradientBoostingClassifier` を使う木系 ML 戦略。
- 非線形性を拾いやすいが、期間が短いと学習が不安定になりやすい。

## worker_06
- `RandomForestClassifier` を使う木系 ML 戦略。
- 相互作用を拾える一方、候補集合が広いとシグナルが拡散しやすい。

## worker_07
- `worker_03` のイベント候補を `worker_04` と `worker_06` の ML で順位付けするハイブリッド戦略。
- 攻撃力が高いが、回転数も多くなりやすい。

## worker_08
- `worker_07` を低回転寄りに圧縮した戦略。
- 取引数は抑えやすいが、最高成績は `worker_07` に届きにくい。

## worker_09
- 中期上昇トレンド中の押し目を買うルールベース戦略。
- 低相関な補完戦略として有効だが、単独の爆発力は高くない。

## worker_10
- イベント候補と押し目候補を統合し、ML とルールスコアを合成する攻撃型戦略。
- 期待値は高いが、乱高下も大きくなりやすい。

## worker_10b
- `worker_10` の守備版。
- 日次採用数や損切りを厳しくして変動を抑える代わりに、成績もやや落とす。

## worker_10c
- `worker_10b` に特徴量距離ベースの分散制約を加えた版。
- 実験上は改善が弱く、採用優先度は低い。

## worker_10d
- `worker_10b` に直近リターン相関ベースの分散制約を加えた版。
- `worker_10` より守備的で、`worker_10b` よりバランスが良い。

## worker_10e
- `worker_10` と `worker_10d` の中間を狙ったブレンド版。
- 実験上は中途半端になりやすく、採用優先度は低い。

## worker_10f
- `worker_10d` にエクスポージャ制御を加えた版。
- 急落日は抑えやすいが、資金投入率を落とすぶん収益力も下がりやすい。

## worker_11
- 低ボラティリティのまま続く中期上昇を追う継続トレンド戦略。
- 静かな上昇には向くが、イベント型ほどの強さは出にくい。

## worker_12
- 明示 train/test 分割で一度だけ学習する短期 split 向け ML ハイブリッド戦略。
- `worker_03` `worker_09` `worker_02` の候補思想をまとめ、短い検証期間でも動きやすいようにしている。

## worker_13
- `worker_07` を主軸に、`worker_03` と `worker_09` の候補が重なる銘柄を加点する合議型メタ戦略。
- 明日買う銘柄選定のような用途を意識し、単独最強よりも再現性と説明しやすさを重視している。

## worker_14
- benchmark に対して相対的に強い銘柄を直接学習する defensive な ML 戦略。
- 指数が弱い日は候補自体を止め、上昇地合いでも benchmark を上回りやすい銘柄だけを狙う。

## worker_15
- benchmark を直接見ずに、低ボラ・浅い押し・小さいギャップの銘柄だけを選ぶ stable compounder 戦略。
- 首位よりも、区間を切っても崩れにくい右肩上がり寄りの成績を狙う。
- 現在は walk-forward で fold ごとに学習し、長期の頑健性を見たうえで直近 fold の良し悪しを運用判断に使う前提。
## worker_16
- live の `meta_consensus` を履歴検証できるようにした backtest 版。
- 各 fold の train 期間で主力候補群の実績を比較し、上位の主力戦略だけを test 期間で合議する。`worker_15b` は安定確認の補助戦略として使う。
## worker_17
- `worker_16` を固定補助型から切り替え、train 期間の直近成績で主力戦略を入れ替える regime-switch 型。
- `worker_15b` は使わず、長期上位かつ直近悪化時に代替しやすい候補群から、その fold 時点で状態の良い戦略だけを採用する。
## worker_17b
- `worker_17` の adaptive 版で、score 差が十分大きい fold は単独主力、僅差 fold は上位2戦略混合、適格なし fold はキャッシュにする。
- 単独化の上振れと、混合の保険の両方を残す regime-switch 型。
