# acceptance_criteria.md

## 1. 共通合格条件
- 参照すべきドキュメントを読んでから着手している
- 役割に対応した成果物が保存されている
- 結果に再現手順または再実行可能なコードがある
- エラー終了時も原因と未完了点が記録されている

## 2. worker 合格条件
- 1つ以上の明確な仮説が言語化されている
- エントリーとイグジットが明文化されている
- `runs/worker_xx/metrics.json` が生成されている
- `runs/worker_xx/result_summary.md` が生成されている
- ベンチマーク比較が含まれている
- 先読み禁止ルールに違反していない
- 手数料・スリッページ考慮の有無が明記されている

## 3. reviewer 合格条件
- 仕様違反の有無が明記されている
- 重大なバイアス懸念が列挙されている
- 再現性や比較可能性への懸念があれば指摘されている
- 是正すべき点と優先度が書かれている

## 4. summary_writer 合格条件
- 全 worker を比較している
- 良い点/悪い点が分離整理されている
- 次回に統合すべき要素が明示されている
- `reports/final_summary.md` が更新されている

## 5. 数値を良く見せるための不合格例
以下に該当する場合は不合格。
- 評価期間の恣意的な切り出し
- 失敗期間の黙殺
- 取引回数が極端に少なく統計的に弱いのに優位と主張
- 実運用困難な前提なのに制約未記載
- ベンチマーク比較なし

## 6. 最低限の保存物
- コードまたは分析ノート
- metrics.json
- result_summary.md
- 必要に応じたログ
